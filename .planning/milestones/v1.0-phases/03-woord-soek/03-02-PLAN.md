  <action>
Create `games/woordsoek/ui.js` as an IIFE exposing `WoordSoekUI` globally. This is the complete UI module — all game interaction in one file.

**State variables:**
- `_words` — filtered word array (loaded once from words.json)
- `_puzzle` — current puzzle object from engine
- `_foundWords` — Set of found word strings
- `_foundCount` — number of found words
- `_selectionState` — 'idle' | 'first_selected'
- `_firstCell` — { row, col } or null
- `_timerInterval` — setInterval ID
- `_elapsedSeconds` — timer counter
- `_currentDifficulty` — 'easy'|'medium'|'hard'
- `_isBelow` — boolean for layout toggle
- `_autoContinue` — boolean for auto-continue setting
- `_gameActive` — boolean to prevent interaction during modals

**Initialization flow:**

`init()` — Called by Router.onEnter. Loads words if not cached (fetch words.json, run WoordSoekEngine.filterWords). Then shows difficulty modal. Reads last difficulty from Settings, pre-highlights that button in modal.

`cleanup()` — Called by Router.onLeave. Stops timer, clears state.

Register lifecycle hooks at IIFE bottom (runs immediately when script loads):
```javascript
Router.onEnter('woordsoek', function() { WoordSoekUI.init(); });
Router.onLeave('woordsoek', function() { WoordSoekUI.cleanup(); });
```

**startGame(difficulty)** — Core game start:
1. Hide difficulty modal (remove 'active' class from ws-difficulty-modal)
2. Save difficulty to Settings: `Settings.set('ws-difficulty', difficulty)`
3. Set `_currentDifficulty = difficulty`
4. Generate puzzle: `WoordSoekEngine.generatePuzzle(_words, difficulty)`
5. Reset state: _foundWords.clear(), _foundCount=0, _selectionState='idle', _firstCell=null, _elapsedSeconds=0, _gameActive=true
6. Set grid CSS: `ws-grid.style.gridTemplateColumns = 'repeat(' + puzzle.cols + ', 1fr)'`
7. Call `renderGrid()` and `renderWordList()`
8. Update counter display
9. Start timer

**renderGrid()** — Build grid cells:
- Clear ws-grid innerHTML
- For each row/col in puzzle.grid, create a div with class 'ws-cell', textContent = letter, data-row and data-col attributes
- Attach click handler on each cell calling `onCellTap(row, col)`
- Attach touchmove handler on ws-grid for preview line (see below)
- Attach touchend handler on ws-grid to clear preview if no second tap

**renderWordList()** — Build word list:
- Clear ws-word-list innerHTML
- For each placed word, create span with class 'ws-word-item', textContent = word
- Sort words alphabetically for display

**onCellTap(row, col)** — Tap-tap selection state machine per user decision:
- If `!_gameActive` return
- If `_selectionState === 'idle'`:
  - Set `_firstCell = {row, col}`, `_selectionState = 'first_selected'`
  - Add 'selected' class to that cell
- If `_selectionState === 'first_selected'`:
  - If same cell tapped: deselect, reset to idle
  - Check direction validity via `WoordSoekEngine.getDirection(_firstCell.row, _firstCell.col, row, col)`
  - If valid direction: call `WoordSoekEngine.checkSelection(...)` with placedWords
  - If word found AND not already in _foundWords:
    - Add to _foundWords, increment _foundCount
    - Call `highlightFoundWord(word, cells)` — set background colour using HIGHLIGHT_COLOURS[_foundCount % 8] on each cell
    - Call `markWordInList(word)` — add 'found' class to matching ws-word-item
    - Update counter: `ws-counter.textContent = _foundCount + ' / ' + _puzzle.placedWords.length`
    - Play sound: `Audio.play('word_found')`
    - Check if all found: if `_foundCount === _puzzle.placedWords.length`, call `onPuzzleComplete()`
    - Reset selection to idle, clear selected/preview classes
    - Return (do NOT treat as new first tap)
  - If word NOT found (invalid direction or no matching word):
    - Clear selected/preview classes on all cells
    - Reset to idle, then IMMEDIATELY treat this tap as new first tap: set `_firstCell = {row, col}`, `_selectionState = 'first_selected'`, add 'selected' class (per user decision: invalid second tap resets silently, new tap becomes first letter)

**Preview line** — As user moves finger after first tap:
- On touchmove on ws-grid: get touch coordinates, find nearest cell using `document.elementFromPoint(touch.clientX, touch.clientY)`
- If element has data-row/data-col, compute direction from _firstCell to that cell via getDirection()
- If valid direction, get cells via getCellsInLine() and add 'preview' class to those cells (clear previous previews first)
- On mousemove: same logic (for desktop testing in Chrome DevTools)

**highlightFoundWord(word, cells)** — For each {r,c} in cells, find the cell element and set its style.background to the current highlight colour. The cell keeps this background permanently (last found wins per user decision — just overwrite).

**Timer:**
- `startTimer()` — `_timerInterval = setInterval(tick, 1000)`. tick: `_elapsedSeconds++; ws-timer.textContent = formatTime(_elapsedSeconds)`
- `stopTimer()` — `clearInterval(_timerInterval)`

**Hints:**
`hint()` — Pick a random word from placedWords that is NOT in _foundWords. Get its first cell {row, col}. Find that cell element. Add 'hint-flash' class (CSS animation defined in Plan 01). Remove class after 1.2s (animation duration). Unlimited hints per user decision.

**Difficulty modal:**
`showDifficultyModal()` — If game is active and _foundCount > 0, show confirm modal first (mid-game). Otherwise show difficulty modal directly. Add 'active' class to ws-difficulty-modal. Pre-highlight the current/last difficulty button with 'ws-diff-selected' class. Set _gameActive = false during modal.

**Confirm new puzzle dialog:**
- When "Nuwe Raaisel" mid-game (via difficulty button in header): show ws-confirm-modal (add 'active' class)
- `confirmNewPuzzle()` — Hide confirm modal, show difficulty modal
- `closeConfirmModal()` — Hide confirm modal, set _gameActive = true

**Win/completion:**
`onPuzzleComplete()` —
1. Stop timer
2. Set _gameActive = false
3. Play `Audio.play('board_finished')`
4. Record stats: `Settings.recordWin('woordsoek', _elapsedSeconds)`
5. Check auto-continue setting `Settings.get('ws-auto-continue', false)`:
   - If true: show toast "Baie goed! " + formatTime(_elapsedSeconds), then after 1.5s call showDifficultyModal() to start new game
   - If false: show ws-win-overlay (add 'active' class). Set ws-win-message text to something like "Jy het alle " + _foundCount + " woorde gevind in " + formatTime(_elapsedSeconds) + "!"
6. Refresh welcome screen stats chip (call `refreshStats()` if available — it's a global function in index.html)

**Layout toggle:**
`toggleLayout()` — Toggle `_isBelow`, toggle 'layout-below' class on ws-layout element. Save preference: `Settings.set('ws-layout-below', _isBelow)`. On init, read preference and apply class if needed.

**Auto-continue setting:**
Read from `Settings.get('ws-auto-continue', false)` on init. No UI toggle for this in the current plan — it's stored in Settings and can be set programmatically or added to settings panel later. Default is false (show win overlay) per user decision.

**Expose:** `return { init, cleanup, startGame, hint, showDifficultyModal, toggleLayout, confirmNewPuzzle, closeConfirmModal }`

**Important implementation notes:**
- All event listeners on the grid should use click events (touch-action:manipulation already on body removes 300ms delay)
- Preview line uses touchmove/mousemove on the grid container, NOT on individual cells
- Use `document.querySelector('[data-row="' + r + '"][data-col="' + c + '"]')` to find cell elements for highlighting
- Do NOT use ESM import/export. IIFE only.
- All UI labels in Afrikaans where shown to user
- Ensure grid cells respect --tap-min (56px) via CSS min-height already set in Plan 01
