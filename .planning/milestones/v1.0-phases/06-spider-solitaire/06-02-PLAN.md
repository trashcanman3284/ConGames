  <name>Task 1: Create Spider UI module (games/spider/ui.js)</name>
  <files>games/spider/ui.js</files>
  <action>
Create `games/spider/ui.js` as an IIFE module following the exact same pattern as `games/solitaire/ui.js`. Use `var SpiderUI = (function() { ... })();` -- NO ESM imports.

**Structure (mirror Solitaire UI):**

1. **State variables:** `_timer`, `_seconds`, `_selectedCard` (null or {col, cardIndex, el}), `_preventClick`, `_isDragging`, `_isAnimating`

2. **DOM cache:** `_els` object populated lazily via `el(id)` helper (same as Solitaire)

3. **init():** Called by Router.onEnter('spider'). Wire all event listeners:
   - Difficulty modal buttons (spd-diff-1suit/2suit/4suit) -> startGame(1/2/4)
   - Undo button -> handleUndo()
   - New game button -> show difficulty modal (or confirm if game in progress)
   - Settings button -> show settings modal
   - Settings toggles -> persist to Settings.set('spider_show_timer', bool), etc.
   - Settings close button -> hideOverlay
   - Win overlay buttons -> new game / Router.go('welcome')
   - Stock pile click -> handleDeal()
   - Show difficulty modal on first entry

4. **cleanup():** Called by Router.onLeave('spider'). Clear timer, reset state, remove event listeners.

5. **startGame(suitMode):**
   - Call SpiderEngine.newGame(suitMode)
   - Hide difficulty modal
   - Reset timer, render board
   - Apply settings visibility
   - Persist last suit mode: Settings.set('spider_suitMode', suitMode)

6. **render():** Full re-render of the board (called after every action):
   - Get state from SpiderEngine.getState()
   - renderTableau(state) -- 10 columns with card stacks
   - renderFoundations(state) -- completed sequences at bottom-left
   - renderStock(state) -- remaining deal piles at bottom-right
   - updateHeader(state) -- timer, moves, score

7. **renderTableau(state):**
   - For each of 10 columns: clear column div, add placeholder if empty
   - For each card in column: create via CardRenderer.createCard(), position with dynamic overlap compression
   - **Overlap compression algorithm** (adapt from Solitaire): Calculate available height = container height - header - bottom bar. Face-down offset: max(6px, calculated). Face-up offset: max(16px, calculated). If total stack height exceeds available, compress proportionally.
   - Add tap handlers to face-up cards: makeCardTapHandler(col, cardIndex)
   - Add drag handlers to face-up cards that start a movable sequence

8. **renderFoundations(state):**
   - Show completed K->A sequences as small card stacks at bottom-left
   - Each foundation shows the top card (King) of the completed sequence
   - Use smaller card scale (~50% or fixed ~40px wide)

9. **renderStock(state):**
   - Show state.dealsRemaining face-down card piles at bottom-right
   - Slightly offset/overlapping to show count visually (MS Spider style)
   - Each pile clickable -> handleDeal()
   - When stock empty, show empty placeholder

10. **Tap-to-move flow** (same state machine as Solitaire):
    - First tap on a face-up card: highlight it and all cards below in its movable sequence (add .selected class). Store in _selectedCard.
    - Second tap on valid destination column: call SpiderEngine.moveCards(from, to). If success, render(). If sequenceCompleted, play sequence animation then render.
    - Second tap on same card: deselect.
    - Tap on invalid target: deselect, show toast.
    - Tap on empty column: valid if card selected.

11. **Drag-and-drop** (clone Solitaire pattern):
    - On pointerdown/touchstart on a face-up card that starts a movable sequence: create floating clone of the sequence cards
    - On pointermove/touchmove: move floating clone to follow pointer
    - On pointerup/touchend: use elementFromPoint to find drop target column. If valid move, execute. Otherwise, snap back.
    - Set _preventClick = true during drag to avoid triggering tap handler.

12. **handleDeal():**
    - Call SpiderEngine.dealFromStock()
    - If success: animate cards flying from stock to each column with staggered delay (~60ms each), then render()
    - If failed with 'empty_column': showToast('Vul alle kolomme eers')
    - If failed with 'empty': showToast('Geen kaarte oor nie')

13. **Sequence completion animation:**
    - When moveCards or dealFromStock returns sequenceCompleted=true (or engine auto-removes):
    - Animate 13 cards flying from their tableau column to the foundation area at bottom-left
    - Play Audio.play('word_found')
    - After animation completes, render() and check isWon()

14. **Win detection and celebration:**
    - After every action, check SpiderEngine.isWon()
    - If won: stop timer, play Audio.play('board_finished'), run win animation (reuse Solitaire's bouncing card trail pattern using spd-win-animation container), show win overlay with time and score
    - Record: Settings.recordWin('spider', _seconds)

15. **Settings visibility** (same as Solitaire):
    - spider_show_timer (default true)
    - spider_show_moves (default true)
    - spider_show_scoring (default true)
    - Apply display:none/inline-block on header elements

16. **Timer:** setInterval counting _seconds, display via formatTime(). Start on first interaction (first move or deal). Pause via clearInterval on cleanup.

**Register Router hooks at module level:**
```javascript
Router.onEnter('spider', function() { SpiderUI.init(); });
Router.onLeave('spider', function() { SpiderUI.cleanup(); });
```

**IMPORTANT:**
- Use `var` not `const`/`let` for max compatibility
- Use CardRenderer.createCard() for ALL card DOM elements -- do NOT create card HTML manually
- Use `spd-` prefix for all DOM ID references
- Do NOT modify any other game's code
