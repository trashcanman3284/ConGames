Create `games/sudoku/engine.js` as an IIFE module following the exact same pattern as `games/solitaire/engine.js` and `games/spider/engine.js`. Use `var SudokuEngine = (function() { ... })();` -- NO ESM imports.

**Constants:**
```javascript
var DIFFICULTY = {
  maklik:  { label: 'Maklik',  revealedCells: 45 },
  medium:  { label: 'Medium',  revealedCells: 35 },
  moeilik: { label: 'Moeilik', revealedCells: 27 },
  kenner:  { label: 'Kenner',  revealedCells: 22 }
};
```

**State structure:**
```javascript
var _state = {
  grid: [],        // 9x9 current values (0 = empty)
  solution: [],    // 9x9 complete solution (for hints/check)
  given: [],       // 9x9 booleans (true = original puzzle cell, not editable)
  notes: [],       // 9x9 arrays, each cell is [false,false,...] (9 booleans for digits 1-9, indexed 0-8)
  difficulty: '',  // 'maklik', 'medium', 'moeilik', 'kenner'
  moves: 0,
  undoStack: []    // array of {type, row, col, prevValue, prevNotes, ...}
};
```

**Public API (return object):**

1. `newGame(difficulty)` -- Generate a new Sudoku puzzle:
   - Step 1: Create empty 9x9 grid (all zeros)
   - Step 2: Fill complete grid using `fillGrid()` backtracking with randomized candidates
   - Step 3: Clone as solution
   - Step 4: Remove cells via `removeCells()` to reach target revealed count
   - Step 5: Clone result as grid, mark given cells, init empty notes, reset undoStack and moves
   - Store difficulty string
   - Return cloned state
   - **Performance:** For 'kenner' difficulty, if generation takes too long (more than ~200 removal attempts without reaching target), accept current state with slightly more revealed cells. This prevents UI freeze on slower devices.

2. `getState()` -- Return deep-cloned state

3. `setValue(row, col, val)` -- Place a number (1-9) or erase (val=0):
   - Refuse if given[row][col] is true (original puzzle cell): return { success: false, reason: 'given' }
   - Push undo record: { type: 'value', row, col, prevValue: grid[row][col], prevNotes: clone of notes[row][col] }
   - Set grid[row][col] = val
   - Clear notes[row][col] entirely (reset all to false)
   - **Auto-clear related notes:** When val > 0, remove that digit from notes in ALL cells sharing the same row, column, and 3x3 box. This is standard MS Sudoku behavior per CONTEXT.md Pitfall 2.
   - Increment moves
   - Return { success: true }

4. `toggleNote(row, col, n)` -- Toggle pencil mark for digit n (1-9):
   - Refuse if given[row][col] is true: return { success: false }
   - Refuse if grid[row][col] !== 0 (cell has a value): return { success: false }
   - Push undo record: { type: 'note', row, col, prevNotes: clone of notes[row][col] }
   - Toggle notes[row][col][n-1] (n is 1-9, index is 0-8)
   - Increment moves
   - Return { success: true }

5. `checkErrors()` -- Return array of {row, col} where grid[row][col] !== 0 AND grid[row][col] !== solution[row][col]. Only checks player-entered cells (given=false).

6. `getHint()` -- Find one empty cell (grid=0, given=false) and return {row, col, value: solution[row][col]}. Pick randomly from empty cells. Return null if no empty cells.

7. `isComplete()` -- Return true when every cell in grid matches solution. Check all 81 cells.

8. `undo()` -- Pop last undo record and reverse:
   - type 'value': restore grid[row][col] = prevValue, restore notes[row][col] = prevNotes. Note: does NOT restore auto-cleared notes in related cells (same as MS Sudoku -- undo only restores the changed cell, not cascading note clears).
   - type 'note': restore notes[row][col] = prevNotes
   - Decrement moves
   - Return { success: true, type } or { success: false } if stack empty

9. `getSavedGame()` -- Return serializable object:
   ```javascript
   { grid, solution, given, notes, difficulty, moves, version: 1 }
   ```
   Does NOT include undoStack (too large to serialize efficiently, acceptable trade-off).

10. `loadGame(savedState)` -- Restore state from saved object:
    - Validate version === 1 and structure has required fields
    - If invalid, return { success: false }
    - Restore grid, solution, given, notes, difficulty, moves
    - Reset undoStack to []
    - Return { success: true }

11. `getCompletedCount(digit)` -- Return count of how many times digit (1-9) appears in grid (both given and player-entered). Used by UI to grey out completed numbers on the pad.

**Internal helpers:**

- `fillGrid(grid)` -- Backtracking: find first empty cell, try shuffled [1-9], recurse. Return true on success. Uses `isValidPlacement(grid, row, col, num)` to check row/col/box constraints.

- `isValidPlacement(grid, row, col, num)` -- Check num not in same row, column, or 3x3 box.

- `removeCells(grid, targetRemoved)` -- Shuffle all 81 cell positions. For each, try removing (set to 0). Run `countSolutions(grid, 2)`. If count === 1, keep removal. If count > 1, restore value. Stop when targetRemoved cells have been removed OR all positions exhausted.

- `countSolutions(grid, limit)` -- Backtracking solver that counts solutions up to `limit` (always pass 2). Early exit as soon as count reaches limit. This is the uniqueness guarantee per SDK-03.

- `shuffle(arr)` -- Fisher-Yates, returns new shuffled array.

- `cloneGrid(grid)` -- Deep clone 9x9 array.

- `cloneNotes(notes)` -- Deep clone 9x9 array of 9-element boolean arrays.

- `getBoxStart(row, col)` -- Return {boxRow, boxCol} for the top-left corner of the 3x3 box containing (row, col). `boxRow = Math.floor(row/3)*3`, `boxCol = Math.floor(col/3)*3`.

**IMPORTANT:**
- Do NOT import or require anything. Pure IIFE, no DOM access.
- Use `var` not `const`/`let` for max Android 10 compatibility.
- The solver `countSolutions` MUST early-exit at limit to avoid exponential blowup on Expert puzzles.
- Notes array uses 0-indexed booleans: notes[r][c][0] = whether digit 1 is noted, notes[r][c][8] = whether digit 9 is noted.
