<task type="auto">
  <name>Task 1: Create Sudoku UI module (games/sudoku/ui.js)</name>
  <files>games/sudoku/ui.js</files>
  <action>
Create `games/sudoku/ui.js` as an IIFE module following the exact same pattern as `games/solitaire/ui.js` and `games/spider/ui.js`. Use `var SudokuUI = (function() { ... })();` -- NO ESM imports.

**Structure:**

1. **State variables:**
   - `_timer` (interval ID), `_elapsedSeconds`, `_timerStarted` (bool)
   - `_selectedCell` (null or {row, col, el})
   - `_notesMode` (bool, false by default)
   - `_manuallyPaused` (bool)
   - `_gameActive` (bool)
   - `_cellEls` (9x9 array of DOM elements for targeted updates)

2. **DOM cache:** `_els` object populated lazily via `el(id)` helper (same as Solitaire/Spider)

3. **init():** Called by Router.onEnter('sudoku'). Wire all event listeners:
   - Difficulty modal buttons -> startGame(difficulty)
   - Resume modal buttons -> resumeGame() / showDifficultyModal()
   - Pause button -> pauseGame()
   - Pause overlay resume button -> resumeGame()
   - Undo button -> handleUndo()
   - New game button -> confirm if game in progress, then show difficulty modal
   - Number pad buttons (via event delegation on sdk-numpad container) -> handleNumInput(digit)
   - Erase button -> handleErase()
   - Notes toggle -> toggleNotesMode()
   - Hint button -> handleHint()
   - Check button -> handleCheck()
   - Win overlay buttons -> new game / Router.go('welcome')
   - `document.addEventListener('visibilitychange', handleVisibilityChange)` for auto-pause
   - On entry: check for saved game via Settings.get('sudoku-save', null). If exists, show resume modal with saved difficulty and elapsed time. If not, show difficulty modal.

4. **cleanup():** Called by Router.onLeave('sudoku'). Clear timer, auto-save if game active, remove visibilitychange listener, reset state.

5. **startGame(difficulty):**
   - Show loading overlay (especially for 'kenner')
   - Use `setTimeout(function() { ... }, 50)` to let the loading overlay render before blocking generation
   - Call SudokuEngine.newGame(difficulty)
   - Hide loading overlay, hide difficulty modal
   - Reset timer, _elapsedSeconds = 0, _timerStarted = false
   - buildGrid() to create 81 cell DOM elements
   - render()
   - Clear any previous saved game: Settings.set('sudoku-save', null)
   - Set _gameActive = true

6. **buildGrid():**
   - Clear sdk-grid innerHTML
   - Create 81 div.sdk-cell elements arranged in 9x9 CSS Grid
   - Each cell: `<div class="sdk-cell" data-row="R" data-col="C"></div>`
   - Apply thick border classes for 3x3 box boundaries (use CSS classes: .sdk-cell-border-right for cols 2,5; .sdk-cell-border-bottom for rows 2,5)
   - Store reference in _cellEls[row][col] for targeted updates
   - Attach click handler to each cell -> handleCellClick(row, col)

7. **render():**
   - Get state from SudokuEngine.getState()
   - For each cell (r,c): call renderCell(r, c, state)
   - Update number pad (grey out completed numbers)
   - Update timer display

8. **renderCell(row, col, state):**
   - Get cell DOM element from _cellEls[row][col]
   - Clear cell content
   - Remove all state classes (sdk-given, sdk-entered, sdk-selected, sdk-highlighted, sdk-error)
   - If state.given[row][col]: add class 'sdk-given', show number in bright white
   - Else if state.grid[row][col] > 0: add class 'sdk-entered', show number in accent gold
   - Else if any notes active: render 3x3 mini-grid of pencil marks
     - Create container div.sdk-notes
     - For each digit 1-9: if notes[row][col][digit-1] is true, show small number in correct position
     - 3x3 layout: digit 1 at position (0,0), 2 at (0,1), 3 at (0,2), 4 at (1,0), ..., 9 at (2,2)
   - Apply highlighting:
     - If cell is selected: add 'sdk-selected' class
     - If cell contains same number as selected cell's number (and number > 0): add 'sdk-highlighted' class
     - If cell is in same row/col/box as selected cell: add 'sdk-related' class (subtle background)

9. **handleCellClick(row, col):**
   - If not _gameActive, return
   - If cell is given, still select it (for highlighting) but don't allow editing
   - Set _selectedCell = {row, col, el: _cellEls[row][col]}
   - Start timer if not started: _timerStarted = true, startTimer()
   - Re-render to update highlighting (selected cell highlight + same-number highlight per CONTEXT.md)

10. **handleNumInput(digit):**
    - If no _selectedCell, show toast 'Kies eers \'n sel'
    - If _notesMode: call SudokuEngine.toggleNote(_selectedCell.row, _selectedCell.col, digit)
    - Else: call SudokuEngine.setValue(_selectedCell.row, _selectedCell.col, digit)
    - If success: render(), autoSave(), check isComplete()
    - If failed with 'given': show toast 'Hierdie sel kan nie verander word nie'

11. **handleErase():**
    - If no _selectedCell, return
    - Call SudokuEngine.setValue(_selectedCell.row, _selectedCell.col, 0)
    - render(), autoSave()

12. **toggleNotesMode():**
    - _notesMode = !_notesMode
    - Update button visual: add/remove 'sdk-notes-active' class on sdk-notes-btn
    - Show toast: _notesMode ? 'Notas modus aan' : 'Notas modus af'

13. **handleHint():**
    - Call SudokuEngine.getHint()
    - If null: show toast 'Geen leë selle nie'
    - Else: call SudokuEngine.setValue(hint.row, hint.col, hint.value)
    - Animate hint cell (brief flash/glow effect)
    - Play Audio.play('word_found')
    - render(), autoSave(), check isComplete()

14. **handleCheck():**
    - Call SudokuEngine.checkErrors()
    - If empty: show toast 'Geen foute!' (per CONTEXT.md)
    - Else: for each error cell, add 'sdk-error' class, then setTimeout to remove after 1500ms (per CONTEXT.md: flash red 1-2 seconds, not persistent)

15. **handleUndo():**
    - Call SudokuEngine.undo()
    - If success: render(), autoSave()
    - If failed: show toast 'Niks om te ontdoen nie'

16. **Timer:**
    - startTimer(): setInterval incrementing _elapsedSeconds every 1000ms, update display via formatTime()
    - pauseTimer(): clearInterval
    - pauseGame(): _manuallyPaused = true, pauseTimer(), show pause overlay with elapsed time (formatTime), autoSave()
    - resumeGame() from pause: _manuallyPaused = false, hide pause overlay, startTimer()
    - resumeGame() from saved: load game via SudokuEngine.loadGame(), restore _elapsedSeconds from saved state, buildGrid(), render(), hide resume modal, _gameActive = true, _timerStarted = true, startTimer()

17. **Auto-pause (visibilitychange):**
    ```javascript
    function handleVisibilityChange() {
      if (!_gameActive) return;
      if (document.hidden) {
        pauseTimer();
        autoSave();
      } else {
        if (!_manuallyPaused) startTimer();
      }
    }
    ```

18. **Auto-save:**
    ```javascript
    function autoSave() {
      if (!_gameActive) return;
      var state = SudokuEngine.getSavedGame();
      state.elapsedSeconds = _elapsedSeconds;
      Settings.set('sudoku-save', state);
    }
    ```

19. **Number pad update:**
    - After each render, for each digit 1-9: call SudokuEngine.getCompletedCount(digit)
    - If count >= 9: add 'sdk-num-completed' class to that button (greyed out per CONTEXT.md)
    - Else: remove 'sdk-num-completed' class

20. **Win detection:**
    - After every setValue or hint: check SudokuEngine.isComplete()
    - If won: _gameActive = false, pauseTimer(), play Audio.play('board_finished')
    - Run win animation (reuse bouncing card trail pattern from Solitaire/Spider but with number elements, or simple confetti using the sdk-win-animation container)
    - Show win overlay with time and difficulty
    - Record: Settings.recordWin('sudoku', _elapsedSeconds)
    - Clear saved game: Settings.set('sudoku-save', null)

21. **Overlay helpers:**
    - showOverlay(id) / hideOverlay(id) -- set display flex/none
    - showDifficultyModal() / hideDifficultyModal()

**Register Router hooks at module level:**
```javascript
Router.onEnter('sudoku', function() { SudokuUI.init(); });
Router.onLeave('sudoku', function() { SudokuUI.cleanup(); });
```

**IMPORTANT:**
- Use `var` not `const`/`let` for max compatibility
- Cell-first input flow: tap cell THEN tap number (per CONTEXT.md locked decision)
- Given numbers in bright white (#f0e6d0 or white), player-entered in accent gold (#d4a23a) (per CONTEXT.md)
- Notes mode is a toggle on the number pad, NOT on individual cells (per CONTEXT.md)
- Pencil marks are display-only within cells (not individually tappable) (per RESEARCH.md pitfall 5)
- Error flash is temporary (1-2 seconds), NOT persistent (per CONTEXT.md)
- Auto-save on EVERY move (per CONTEXT.md locked decision)
- Use sdk- prefix for all DOM ID references
- Do NOT modify any other game's code
