  <name>Task 1: Create FreeCell UI module (games/freecell/ui.js)</name>
  <files>games/freecell/ui.js</files>
  <action>
Create `games/freecell/ui.js` as an IIFE module following the exact same pattern as `games/solitaire/ui.js` and `games/spider/ui.js`. Use `var FreeCellUI = (function() { ... })();` -- NO ESM imports.

**Structure (mirror Solitaire/Spider UI):**

1. **State variables:** `_timer`, `_seconds`, `_selectedCard` (null or {zone, col, cardIndex, el, cards}), `_preventClick`, `_isDragging`, `_isAnimating`, `_gameStarted`

2. **DOM cache:** `_els` object populated lazily via `el(id)` helper (same as Solitaire/Spider)

3. **init():** Called by Router.onEnter('freecell'). Wire all event listeners:
   - Undo button -> handleUndo()
   - Restart button -> handleRestart() (confirms if game in progress, then FreeCellEngine.restart())
   - New game button -> handleNewGame() (confirms if game in progress, then FreeCellEngine.newGame())
   - Settings button -> show settings modal
   - Settings toggles -> persist to Settings.get/set('freecell_show_timer', bool), etc.
   - Settings close button -> hideOverlay
   - Win overlay buttons -> new game / Router.go('welcome')
   - Start new game immediately on entry (no difficulty modal needed -- FreeCell has no difficulty settings)

4. **cleanup():** Called by Router.onLeave('freecell'). Clear timer, reset state, remove event listeners.

5. **startGame(dealNumber):**
   - Call FreeCellEngine.newGame(dealNumber) -- undefined for random
   - Reset timer, render board
   - Update deal number display: "Spel #" + FreeCellEngine.getDealNumber()
   - Apply settings visibility

6. **render():** Full re-render of the board (called after every action):
   - Get state from FreeCellEngine.getState()
   - renderFreeCells(state) -- 4 slots top-left
   - renderFoundations(state) -- 4 piles top-right
   - renderTableau(state) -- 8 columns
   - updateHeader(state) -- timer, moves, deal number

7. **renderFreeCells(state):**
   - For each of 4 free cell slots: clear div
   - If null: show dashed/outlined rectangle placeholder (CSS class `fc-empty-cell`)
   - If card: render via CardRenderer.createCard(), add tap handler
   - Tap handler on free cell card: select it (zone: 'freecell', col: N)

8. **renderFoundations(state):**
   - For each of 4 foundation piles: clear div
   - If empty: show faint suit symbol placeholder (use suit symbols in faded text or CSS class `fc-empty-found`)
   - If cards: show only top card via CardRenderer.createCard()
   - Foundation piles are NOT tap targets for source (only destinations)

9. **renderTableau(state):**
   - For each of 8 columns: clear column div, add placeholder if empty (tap target for receiving cards)
   - For each card in column: create via CardRenderer.createCard(), position with dynamic overlap compression
   - **Overlap compression algorithm** (adapt from Solitaire/Spider): Calculate available height. Face-up offset (all cards are face-up in FreeCell): max(20px, calculated). If total stack height exceeds available, compress proportionally.
   - Add tap handlers to all cards: makeCardTapHandler(col, cardIndex)
   - Add drag handlers

10. **Tap-to-move flow (3 modes per user decision):**

    a) **First tap on a card (tableau or freecell):**
       - If tableau: auto-detect largest valid descending alternating-colour sequence from that card down. Highlight the whole group with .selected class. Store in _selectedCard with zone, col, cardIndex, cards array.
       - If freecell: highlight single card. Store in _selectedCard.

    b) **Second tap (card already selected):**
       - Tap on same card or same group: this is the "double-tap" auto-move. Try auto-move to foundation first (FreeCellEngine.moveCards to foundation). If no valid foundation, try moving to first empty free cell. If neither works, deselect.
       - Tap on valid destination (tableau column, empty column, free cell, foundation): call FreeCellEngine.moveCards(from, to). If success, render(). If autoMoves > 0, animate auto-foundation moves before final render.
       - Tap on invalid target: trigger **shake animation** on selected card(s) (add .shake CSS class, remove after 400ms). Deselect. No sound per user decision.

    c) **Tap on empty column (no card selected):** do nothing.
       **Tap on empty column (card selected):** move selected card(s) there.
       **Tap on empty free cell (card selected):** move single card there.
       **Tap on foundation pile (card selected):** move single card there if valid.

11. **Drag-and-drop** (clone Solitaire/Spider pattern):
    - On pointerdown/touchstart on a card: create floating clone of card(s)
    - On pointermove/touchmove: move floating clone to follow pointer
    - On pointerup/touchend: use elementFromPoint to find drop target. Determine target zone (freecell, foundation, tableau column). If valid move, execute. Otherwise, snap back.
    - Set _preventClick = true during drag to avoid triggering tap handler.

12. **Auto-foundation animation** (per user decision: ~200ms fly animation):
    - When moveCards returns autoMoves > 0: for each auto-moved card, animate it flying from its source position to the correct foundation pile.
    - Use absolute positioning + CSS transition or requestAnimationFrame.
    - Stagger multiple auto-moves (~200ms apart).
    - After all animations complete, do final render().

13. **Shake animation on invalid move:**
    - Add CSS @keyframes shake to shared.css (see Task 2).
    - On invalid move: add .shake class to all selected card elements. Remove after 400ms via setTimeout. Deselect.

14. **Win detection and celebration:**
    - After every action + auto-foundation, check FreeCellEngine.isWon()
    - If won: stop timer, play Audio.play('board_finished'), run win animation (reuse Solitaire's bouncing card trail pattern using fc-win-animation container), show win overlay with time, moves, and deal number ("Spel #N" per user decision)
    - Record: Settings.recordWin('freecell', _seconds)

15. **Settings visibility** (same as Solitaire/Spider):
    - freecell_show_timer (default true)
    - freecell_show_moves (default true)
    - Apply display:none/inline-block on header elements

16. **Timer:** setInterval counting _seconds, display via formatTime(). Start on first move (_gameStarted flag). Pause via clearInterval on cleanup.

17. **Restart (Herbegin):** Call FreeCellEngine.restart(), reset timer, render. Show in win overlay too.

**Register Router hooks at module level:**
```javascript
Router.onEnter('freecell', function() { FreeCellUI.init(); });
Router.onLeave('freecell', function() { FreeCellUI.cleanup(); });
```

**IMPORTANT:**
- Use `var` not `const`/`let` for max compatibility
- Use CardRenderer.createCard() for ALL card DOM elements -- do NOT create card HTML manually
- Use `fc-` prefix for all DOM ID references
- Do NOT modify any other game's code
- Invalid moves: shake + deselect, NO sound, NO toast (per user decision)
- Double-tap = tap already-selected card = auto-move (foundation first, then free cell)
- Deal number visible in header AND on win screen
