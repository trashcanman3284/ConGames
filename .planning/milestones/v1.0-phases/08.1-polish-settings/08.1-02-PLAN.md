  <action>
Modify the WoordSoekUI IIFE to add drag-to-select alongside existing tap-tap. Both must work simultaneously.

1. Add new state variables at the top of the IIFE (alongside existing state):
   ```
   var _isDragging = false;
   var _dragStartCell = null;
   var _lastDragRow = null;
   var _lastDragCol = null;
   var _justCompletedDrag = false;
   ```

2. In `renderGrid()`, add a `touchstart` listener on the grid (alongside existing touchmove/touchend):
   ```
   grid.addEventListener('touchstart', onGridTouchStart, { passive: true });
   ```

3. Create `onGridTouchStart(e)`:
   - Get touch point via `e.touches[0]`
   - Use `document.elementFromPoint()` to find target cell
   - If target has `.ws-cell` class, store row/col in `_dragStartCell`
   - Set `_isDragging = false` (not yet -- needs movement threshold)
   - Set `_lastDragRow = null`, `_lastDragCol = null`

4. Modify `onTouchMove(e)` (the existing function):
   - Keep the existing behaviour for `_selectionState === 'first_selected'` (preview during tap-tap)
   - Add NEW logic: if `_dragStartCell` is set, get current cell under finger
   - Calculate distance: `Math.max(Math.abs(r - _dragStartCell.row), Math.abs(c - _dragStartCell.col))`
   - If distance >= 2, set `_isDragging = true`
   - If `_isDragging`:
     - If `_firstCell` is not set from drag yet, set `_firstCell = _dragStartCell` and `_selectionState = 'first_selected'` and add 'selected' class to the start cell
     - Call `showPreviewTo(touch.clientX, touch.clientY)` (existing function handles the rest)
     - Store current cell in `_lastDragRow` and `_lastDragCol`
   - Use `{ passive: true }` on the listener (already set in renderGrid)

5. Rewrite `onTouchEnd(e)`:
   - If `_isDragging && _firstCell && _lastDragRow !== null`:
     - Check direction: `WoordSoekEngine.getDirection(_firstCell.row, _firstCell.col, _lastDragRow, _lastDragCol)`
     - If valid direction, call `WoordSoekEngine.checkSelection(...)` with first cell and last drag cell
     - If word found and not already in `_foundWords`:
       - Add to `_foundWords`, increment `_foundCount`
       - Call `highlightFoundWord()`, `markWordInList()`
       - Update counter, play `Audio.play('word_found')`
       - Check completion
     - If no word found: silent clear (no shake, no sound -- per locked decision)
     - Call `clearSelection()`, `clearPreview()`
     - Set `_justCompletedDrag = true`
     - `setTimeout(function() { _justCompletedDrag = false; }, 400)` to prevent ghost click
   - If NOT dragging: just call `clearPreview()` (preserve existing behaviour for tap-tap)
   - Reset: `_isDragging = false`, `_dragStartCell = null`, `_lastDragRow = null`, `_lastDragCol = null`

6. Modify `onCellTap(row, col)`:
   - Add at the very top: `if (_justCompletedDrag) return;` — prevents ghost click after drag (Pitfall 1 from RESEARCH.md)
   - Rest of tap-tap logic stays exactly the same

7. In `cleanup()`, reset the new state variables:
   ```
   _isDragging = false;
   _dragStartCell = null;
   _lastDragRow = null;
   _lastDragCol = null;
   _justCompletedDrag = false;
   ```

IMPORTANT:
- Do NOT call `e.preventDefault()` in touchstart -- it breaks click fallback for tap-tap (Anti-pattern from RESEARCH.md)
- The touchmove listener MUST remain `{ passive: true }` since we don't call preventDefault
- Use `.closest('.ws-cell')` or check `.classList.contains('ws-cell')` on elementFromPoint result (Pitfall 2)
- The existing `showPreviewTo()` function already handles direction snapping and cell highlighting -- reuse it, don't rebuild
