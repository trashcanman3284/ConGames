# Phase 08.1: Polish & Settings - Context

**Gathered:** 2026-03-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Add drag-to-select for Woord Soek, MS-style colour theme backgrounds for all card games (classic green felt + MS card backs), parchment background for Woord Soek, font size setting on welcome screen with per-game integration, and standardized settings modals across all 5 games. Sudoku gets a new settings modal. Woord Soek gets a settings modal to match the others.

</domain>

<decisions>
## Implementation Decisions

### Drag-to-select (Woord Soek)
- Both drag-to-select AND tap-tap work simultaneously — Dad uses whichever feels natural
- During drag: semi-transparent preview highlight along the valid line (enhance existing preview CSS)
- Drag snaps to nearest valid direction (8 compass directions) once past ~2 cells
- Invalid drag (no word found on lift): silent clear, no shake, no sound — just resets
- Existing touchmove/touchend handlers already present — extend them to complete selection on touchend

### MS Colour Themes
- Classic green felt background for ALL card games (Solitaire, Spider, FreeCell) — the iconic MS look
- MS theme is the DEFAULT — warm-dark becomes the alternative
- Card backs change to classic MS blue crosshatch/castle pattern
- Woord Soek gets a parchment/cream background — old word puzzle book feel, fits Afrikaans heritage
- Sudoku keeps current warm-dark theme (not an MS game reference)
- No theme toggle needed for v1 — MS theme is just the default, warm-dark is removed as default

### Font Size Integration
- Font size control lives on the welcome screen ONLY — one global setting, applies everywhere
- Three labelled buttons: "Klein" / "Medium" / "Groot" — tap to select, highlighted state
- Text only scales (button labels, headers, word lists, number pads) — cards and grid cells stay fixed size
- Sudoku: both number pad AND pencil marks scale with font size
- Existing `Settings.applyFontSize()` already sets root font-size (16/18/21px) — just needs UI control + ensuring games use rem/em units

### Settings UI Consistency
- ALL 5 games get a standardized settings modal with gear icon (⚙) button, no text label
- Same modal width, same toggle style, same close button position, same section headers across all games
- Sudoku: NEW settings modal with toggles for timer, highlight same numbers, show errors
- Woord Soek: NEW settings modal consolidating timer, auto-continue, layout position toggles
- Solitaire/Spider/FreeCell: existing modals already follow a consistent pattern — minor alignment tweaks if needed
- Game-specific toggles vary but the container/styling is identical

### Claude's Discretion
- Exact green felt CSS pattern (gradient, texture, or solid colour)
- MS card back visual implementation (CSS pattern or simplified version)
- Parchment background exact colour/texture for Woord Soek
- Modal animation and transition details
- How to handle font size with card corner text (rank/suit)

</decisions>

<specifics>
## Specific Ideas

- MS Solitaire green felt is THE reference — Dad will recognise it instantly
- Card backs should evoke the classic blue crosshatch pattern from Windows Solitaire
- Parchment for Woord Soek should feel like a physical word puzzle book
- Font size buttons should be Afrikaans: "Klein" / "Medium" / "Groot"
- Drag-to-select should feel instant — no delay, no confirmation step

</specifics>
