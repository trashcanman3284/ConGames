<task type="auto">
  <name>Task 1: Solitaire UI module — rendering, interaction, and game flow</name>
  <files>games/solitaire/ui.js</files>
  <action>
Create `games/solitaire/ui.js` as a `var SolitaireUI = (function() { ... })();` IIFE following the WoordSoekUI pattern.

**Module structure:**

```
State variables:
  _els = {}                  // DOM cache
  _timerInterval = null      // Timer
  _elapsedSeconds = 0
  _selectedCard = null       // { zone, col, cardIndex, el }
  _isAutoCompleting = false  // Block input during auto-complete
  _isDragging = false
  _dragState = null          // { startX, startY, cards, originZone, originCol, originCardIndex, floatEl }
  _gameStarted = false       // Has at least one move been made (for loss tracking)
```

**Core rendering — `render()` function:**
Re-render the entire board from SolitaireEngine.getState(). This is simpler and safer than incremental updates.

1. **Stock pile:** Clear `sol-stock`. If stock has cards, render a single face-down card. If empty, render a placeholder (clickable to recycle). Add tap handler.

2. **Waste pile:** Clear `sol-waste`. Render top 1 card face-up (or top 3 spread slightly if draw-3 mode, showing the 3 most recently drawn). Only the top card is interactive.

3. **Foundations (sol-f0 to sol-f3):** Clear each. If empty, render placeholder. If has cards, render only the top card face-up. Add tap handler for destination.

4. **Tableau (sol-col0 to sol-col6):** Clear each column. For each card in column, use `CardRenderer.createCard(rank, suit, faceUp)`. Apply the `.card-stack` pattern. Face-up cards get tap and touch handlers. Set `data-zone="tableau"`, `data-col="N"`, `data-card-index="N"` on each card element for event delegation.

5. **Dynamic column overlap:** If a column has more than 8 cards, calculate a tighter margin-top so cards fit within the available vertical space. Apply via inline style `card.style.marginTop = -X + '%'`. Available height = viewport height - header height (~56px) - top row height (~200px) - padding.

6. **Header elements:** Update timer display (`formatTime`), move counter (`N skuiwe`), score display (`N punte`). Show/hide based on settings toggles.

**Selection and move system — tap-tap interaction:**

1. `handleCardTap(zone, col, cardIndex)` — Main tap handler.
   - If nothing selected: select this card (add `.selected` class). If destination highlighting enabled (`sol-show-hints` setting), call `SolitaireEngine.getValidMoves()` and add `.valid-target` class to valid destination piles/columns.
   - If something already selected and this is a valid destination: call `SolitaireEngine.moveCards(from, to)`. On success: `render()`, play sound (`Audio.play('word_found')` for foundation moves), check for auto-complete. On fail: show toast "Ongeldige skuif".
   - If tapping same card: deselect.
   - If tapping different card that isn't a valid destination: deselect current, select new card.

2. `handleStockTap()` — If stock has cards: `SolitaireEngine.drawFromStock()`. If stock empty: `SolitaireEngine.recycleWaste()` with brief visual sweep (0.3s CSS transition). Then `render()`.

3. `handleFoundationTap(foundIndex)` — If card selected, try `moveCards` to this foundation. If nothing selected and foundation has cards, select top foundation card (for moving back to tableau).

4. After each successful move, check `result.autoComplete`. If true, start auto-complete sequence.

**Auto-move on single tap (no second tap needed):**
If user taps a card and there is exactly one valid destination AND it's a foundation, auto-move it there immediately (no need for second tap). Use `canAutoMoveToFoundation()` to determine safety.

**Drag-and-drop interaction:**

1. On `touchstart` on a face-up card: record start position, set up potential drag.
2. On `touchmove`: if finger moved > 10px from start, enter drag mode:
   - Set `_isDragging = true`
   - Create a floating container (`position:absolute; z-index:50; pointer-events:none;`) with clones of the card (and stack below it if stack move)
   - Move float element to follow finger position (use `e.touches[0].clientX/Y`)
   - Add `.dragging` class to original cards (opacity: 0.3)
3. On `touchend`:
   - If was dragging: find drop target using `document.elementFromPoint(x, y)` (temporarily hide float element)
   - Check if drop target is a valid destination (look for `data-zone`, `data-col` attributes, or foundation/column container IDs)
   - If valid: execute move via engine, render
   - If invalid: animate float back to origin (0.2s transition), then remove
   - Clean up: remove float element, remove `.dragging` class, reset drag state
4. Prevent `click` event when drag occurred (set flag, clear after timeout).

**Timer:**
Start on first move. `setInterval` every 1000ms incrementing `_elapsedSeconds`. Display via `formatTime()`. Pause when leaving screen (`Router.onLeave`). Resume on return.

**Settings integration:**
All settings use `Settings.get/set` with `sol-` prefix:
- `sol-draw-mode` (1 or 3) — persisted, shown in draw modal
- `sol-show-hints` (true/false) — destination highlighting
- `sol-show-timer` (true/false) — timer visibility
- `sol-show-moves` (true/false) — move counter visibility
- `sol-scoring` (true/false) — scoring enabled

Settings modal wiring:
- Open on sol-settings-btn click
- Each toggle reads/writes Settings and immediately shows/hides the relevant header element
- Close on sol-settings-close click

**Game lifecycle:**

`init()`:
1. Cache DOM elements
2. Wire event listeners (undo btn, new game btn, settings btn, draw modal buttons, win overlay buttons)
3. Show draw mode modal (or use last preference from Settings)
4. Start new game with selected draw mode

`startGame(drawMode)`:
1. If `_gameStarted`, record loss: `Settings.recordLoss('solitaire')`, `refreshStats()`
2. `SolitaireEngine.newGame(drawMode)`
3. Reset timer, moves display
4. `render()`
5. Hide modals
6. Set `_gameStarted = false` (becomes true on first move)

`cleanup()`:
1. Clear timer interval
2. Remove event listeners
3. Clear DOM cache

**Undo:**
Call `SolitaireEngine.undo()`. If success, `render()`. If undoStack empty, show toast "Niks om te ontdoen".

**New Game button:**
Call `startGame()` with current draw mode. No confirmation dialog per CONTEXT.md ("no confirmation dialog"). Counts as loss.

**Router hooks (at bottom of file):**
```javascript
Router.onEnter('solitaire', function() { SolitaireUI.init(); });
Router.onLeave('solitaire', function() { SolitaireUI.cleanup(); });
```

<task type="auto">
  <name>Task 2: Auto-complete animation and Windows-style win celebration</name>
  <files>games/solitaire/ui.js, css/cards.css</files>
  <action>
Add to the SolitaireUI IIFE (inside the same file created in Task 1 — these are part of the same module, just described separately for clarity):

**Auto-complete animation — `startAutoComplete()`:**
1. Set `_isAutoCompleting = true`. Disable undo button, new game button.
2. Run a loop with `setInterval(80)` (80ms between each card move):
   - Call `SolitaireEngine.autoCompleteStep()`
   - If returns { card, foundIndex }: animate the card moving to the foundation (brief CSS transition), then `render()` to update state
   - If returns null: auto-complete done, clear interval
3. After auto-complete finishes, check `SolitaireEngine.isWon()`. If yes, trigger win sequence.

**Win sequence — `showWin()`:**
1. Stop timer
2. Play `Audio.play('board_finished')`
3. Record win: `Settings.recordWin('solitaire', _elapsedSeconds)`, `refreshStats()`
4. Run cascading card animation for 4 seconds, then show win overlay

**Windows cascading card animation — `runWinAnimation()`:**
Implementation in the `sol-win-animation` container (absolute positioned, covers game area):

1. Show `sol-win-animation` container (`display:block`)
2. For each foundation (4 total), get all 13 cards. Launch cards one at a time with staggered timing (every 150ms, cycling through foundations).
3. For each launched card:
   - Create a card element using `CardRenderer.createCard(rank, suit, true)`
   - Position it absolutely at the foundation's screen position
   - Give it random horizontal velocity (vx = (Math.random() - 0.5) * 10)
   - Give it upward initial velocity (vy = -(Math.random() * 6 + 2))
   - Apply gravity (0.4 per frame)
   - On each `requestAnimationFrame`:
     - Update position: x += vx, y += vy, vy += gravity
     - Bounce off bottom: if y > containerHeight - cardHeight, reverse vy with 0.7 damping
     - Bounce off sides: if x < 0 or x > containerWidth - cardWidth, reverse vx
     - **Trail effect:** Clone the card element at current position and append to container (do NOT remove previous positions). This creates the classic Windows Solitaire paint trail.
     - Stop individual card when it has bounced 5+ times or after 4 seconds
   - Limit concurrent bouncing cards to ~20 for performance
4. After 4 seconds total, stop animation and show win overlay

5. Win overlay shows:
   - "Geluk!" heading
   - Stats message: time, moves, score (if scoring enabled)
   - "Nuwe Spel" button (big, primary) — calls `startGame`
   - "Tuis" button (small, secondary) — calls `Router.go('welcome')`

**Card trail cleanup:** When hiding the win animation container, set its innerHTML to '' to remove all trail elements.

**Add to css/cards.css:**

```

**Dynamic column compression — add to css/cards.css:**

  /* Overridden by inline marginTop in JS */
}
```

No additional CSS is strictly needed since the dynamic overlap is handled via inline styles in JS. The `.valid-target` glow is defined in the Plan 01 HTML styles.
