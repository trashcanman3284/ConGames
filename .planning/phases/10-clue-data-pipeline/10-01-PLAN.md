    - words.json (cross-reference source)
  </read_first>
  <action>
1. Read `/tmp/selected.json` to get the ~300 word+gloss pairs.

2. For each entry, translate the English gloss to a short Afrikaans clue following these rules:
   - Max 8 Afrikaans words per clue
   - Do NOT use the answer word or any word sharing its first min(wordLength, 5) letters in the clue
   - Mix of styles: some straightforward definitions ("Troue viervoetige huisdier" for HOND), some evocative hints ("Buite kook oor kole" for BRAAI)
   - Accessible to an ordinary Afrikaans adult — no specialist vocabulary
   - SA cultural flavor where appropriate (braai, rugby, rooibos, biltong references)
   - Categories to use naturally: animals, food, household items, actions, nature, sports, family, church, weather, garden

3. Root detection check — for each translated clue, verify:
   ```javascript
   function rootInClue(word, clue) {
     const root = word.slice(0, Math.min(word.length, 5)).toLowerCase();
     return new RegExp('\\b' + root, 'i').test(clue);
   }
   ```
   If root detected, rewrite the clue to avoid it.

4. Write the final `games/kruiswoord/clues.json` as:
   ```json
   [
     { "word": "HOND", "clue": "Troue viervoetige huisdier" },
     ...
   ]
   ```
   Sorted alphabetically by word. Entries have exactly two fields: `word` (uppercase) and `clue` (Afrikaans string).

5. Write a throwaway validation script `/tmp/validate.js` that checks ALL rules:
   - Every entry has "word" and "clue" string fields
   - word matches `/^[A-Z]{4,10}$/`
   - word.toLowerCase() exists in words.json (loaded as Set)
   - clue has at most 8 words (`clue.trim().split(/\s+/).length <= 8`)
   - root NOT in clue (using rootInClue function above)
   - Total entries >= 250
   - Length distribution report: short/medium/long counts and percentages

6. Run: `node /tmp/validate.js`
   Fix any failures — rewrite offending clues and re-validate until 0 errors.

7. Ensure the `games/kruiswoord/` directory exists: `mkdir -p games/kruiswoord`
