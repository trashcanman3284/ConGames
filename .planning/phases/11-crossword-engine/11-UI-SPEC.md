# Phase 11 — UI Design Contract: Crossword Engine

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 11 is a pure logic engine — no DOM, no rendering. The visual contract
> defined here specifies the data structures the engine MUST emit so Phase 12 UI can render
> without re-designing. This spec is the bridge between engine output and visual implementation.

## Spacing Scale

Uses existing project token set from `css/shared.css`. These values are already declared — do not redefine.

| Token | Value | Usage |
|-------|-------|-------|
| `--gap-xs` | 6px | Icon gaps, cell number padding |
| `--gap-sm` | 12px | Clue list item padding, compact element spacing |
| `--gap-md` | 20px | Default section spacing |
| `--gap-lg` | 32px | Layout section gaps |
| `--gap-xl` | 48px | Major section breaks |

### Exceptions

The following tokens are outside the standard 4/8/16/24/32/48/64 set. They are pre-existing project tokens from `css/shared.css` and must not be changed — altering them would break backwards compatibility across 5 already-shipped games.

| Token | Value | Deviation | Justification |
|-------|-------|-----------|---------------|
| `--gap-xs` | 6px | Not a multiple of 4 | Pre-existing token used throughout Woord Soek, Solitaire, Spider, Sudoku, FreeCell UIs |
| `--gap-sm` | 12px | Not in standard set | Pre-existing token; changing it breaks clue and card padding across all shipped games |
| `--gap-md` | 20px | Not in standard set | Pre-existing token; used as the default section spacing token in all game layouts |

Additional exceptions:
- Crossword grid cells: size is **computed** from available viewport height, not a fixed token. Grid must be square and fill available height. Each cell = `floor(gridContainerHeight / gridSize)` px. Minimum cell size: 32px on 17×17. Maximum: uncapped.
- Superscript cell numbers (8px) are an internal grid rendering detail, not a layout spacing value — excluded from the spacing scale.

## Typography

**Declared type scale: 4 sizes, 2 weights.**

| Role | Size | Weight | Line Height | CSS | Source |
|------|------|--------|-------------|-----|--------|
| Body / clue list | 18px (1rem, base size) | 400 | 1.5 | `var(--font-body)` | `css/shared.css` html `font-size: 18px` |
| Heading / section title (Reg/Af) | ~22px (1.2rem) | 700 | 1.2 | `var(--font-body)` weight 700 | Established game header pattern |
| Display / game title | ~25px (1.4rem) | 700 | 1.0 | `var(--font-display)` | `.game-header h1` in shared.css |
| Cell entered letter | `clamp(1rem, 2.5vw, 1.6rem)` (18–29px) | 700 | 1.0 | `var(--font-body)` | Must be legible at minimum 32px cell size |

> **Weight 400** is used for body/clue list text only.
> **Weight 700** is used for all headings, the game title, and cell letters.
> Weight 600 is not used. "Label / clue number prefix" rows in the clue list are distinguished from body text via `--accent-gold` colour, not a third weight.

Crossword-specific typography note (internal grid rendering — excluded from type scale):
- **Cell superscript number**: `var(--font-body)`, weight 400, size fixed at **8px**, positioned `top: 1px; left: 1px; line-height: 1`. This value is below the layout type scale and is a grid-internal rendering detail only.

## Color

Uses existing project palette from `css/shared.css`. All values are already declared.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--bg-base` | `#1a1610` | App background, black crossword cells |
| Secondary (30%) | `--bg-surface` | `#252018` | Header bar, clue panel background |
| Accent (10%) | `--accent-gold` | `#d4a23a` | Reserved for specific elements only |
| Semantic: success | `--green-found` | `#4caf78` | Completed word flash only |
| Semantic: error | `--red-error` | `#e05757` | Not used in Phase 11 engine output |

Accent (`--accent-gold`) reserved for:
- Selected cell border ring
- Active (highlighted) word background tint
- Cell number superscripts on selected word
- Clue list item — active/selected state highlight
- Grid outer border
- Clue number prefix label (distinguishes label from body weight, not weight)

> **Phase 12 note:** The `Verlaat raaisel` button in the back-confirmation modal should be styled with `--red-error` or a derived destructive variant — not `btn-primary`. This is a destructive action (discards progress) and must not share the same visual weight as a constructive primary action.

Crossword cell color mapping (engine data → Phase 12 CSS):
- `isBlack: true` → background `var(--bg-base)` (`#1a1610`), no border
- `isBlack: false` (white cell, unselected) → background `#ffffff`, color `var(--bg-base)` for entered text
- `isBlack: false` (selected word, not active cell) → background `rgba(212,162,58,0.20)` (gold tint)
- `isBlack: false` (active cell — cursor position) → background `rgba(212,162,58,0.45)` (strong gold tint), border `2px solid var(--accent-gold)`
- `complete: true` word cells → flash `var(--green-found)` background for 600ms, then lock to `rgba(76,175,120,0.15)` tint

**Primary focal point:** The crossword grid, occupying the upper 70% of the viewport. The clue list panel sits below the grid in portrait and to the side in landscape. All layout decisions in Phase 12 must preserve the grid's dominance — it is the primary interactive surface.

## Engine Output Contract

> This section is the authoritative data contract between Phase 11 (engine) and Phase 12 (UI).
> Phase 12 must NOT define its own data structures — it consumes exactly what is defined here.

### Cell Object (`grid[row][col]`)

```javascript
{
  letter:  String,   // Correct answer letter, A-Z uppercase. '' if black cell.
  number:  Number,   // 1..N if this cell starts an Across or Down word. 0 otherwise.
  isBlack: Boolean,  // true = black filler cell, false = white playable cell
  entered: String    // What Con has typed. '' = empty, 'H' = entered. Always uppercase.
}
```

### Word Object (`words[]`)

```javascript
{
  word:      String,   // Correct answer, e.g. 'HOND'
  clue:      String,   // Afrikaans clue text, e.g. 'Troue viervoetige huisdier'
  number:    Number,   // Cell number where this word starts (matches grid[row][col].number)
  direction: String,   // 'across' or 'down' — lowercase only
  row:       Number,   // 0-indexed row of first cell
  col:       Number,   // 0-indexed column of first cell
  length:    Number,   // Character count
  complete:  Boolean   // true when all entered letters match word. Engine sets this via checkWord().
}
```

### `generate(difficulty, clues)` return value

```javascript
{
  grid:  Array,  // 2D array [row][col] of cell objects (size × size)
  words: Array   // flat array of word objects, all directions combined
}
```

Phase 12 UI groups words by direction: `words.filter(w => w.direction === 'across')` / `'down'`.

### Public API surface

| Method | Signature | Returns | Side effect |
|--------|-----------|---------|-------------|
| `generate` | `(difficulty: string, clues: Array)` | `{grid, words}` (deep clone via `getState`) | Resets `_state` |
| `setLetter` | `(row: number, col: number, letter: string)` | `void` | Mutates `_state.grid[row][col].entered`, pushes to undo stack |
| `checkWord` | `(number: number, direction: string)` | `{correct: boolean, letters: Array<{row, col, correct}>}` | Sets `word.complete = true` if correct |
| `isComplete` | `()` | `boolean` | None |
| `undo` | `()` | `{success: boolean}` | Pops undo stack, reverts one `setLetter` |
| `getState` | `()` | Deep-cloned `{grid, words, difficulty, elapsedSeconds}` | None |
| `getElapsed` | `()` | `number` (seconds, integer) | None |

`DIFFICULTY` constant exposed on the engine object:

```javascript
KruiswoordEngine.DIFFICULTY = {
  maklik:  { label: 'Maklik',  size: 9,  wordCount: 7  },
  medium:  { label: 'Medium',  size: 13, wordCount: 13 },
  moeilik: { label: 'Moeilik', size: 17, wordCount: 18 }
}
```

## Interaction States

> Engine internal states that Phase 12 UI must handle. Engine emits data; UI reacts visually.

| State | Engine signal | Phase 12 visual |
|-------|--------------|-----------------|
| Cell empty | `entered: ''` | White cell, no letter rendered |
| Cell filled (unverified) | `entered: 'X'` | Letter displayed in `var(--accent-gold)` weight 700 |
| Word correct | `checkWord()` returns `{correct: true}` | Flash green (`--green-found`) × 600ms, lock cells |
| Word incorrect | `checkWord()` returns `{correct: false}` | No visual change — UI does not auto-check; checking triggered only on word completion |
| Puzzle complete | `isComplete()` returns `true` | Congratulations modal (`win-overlay` + `win-card` from shared.css) |
| Generation failure (sparse grid) | `words.length < config.wordCount` after 5 retries | Engine returns best attempt — Phase 12 renders it as-is, no error state shown to Con |

## Destructive Actions

| Action | Trigger | Confirmation approach |
|--------|---------|----------------------|
| Leave in-progress puzzle | Tap back button while `isComplete()` is false | Inline confirmation modal: two buttons, no toast. `Bly hier` = stay. `Verlaat raaisel` = leave and discard. |
| New game while playing | `Nuwe Raaisel` tap (from win modal only) | No confirmation — win modal only appears after completion, puzzle is already done |
